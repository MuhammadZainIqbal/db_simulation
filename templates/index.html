About This Project

The Technical Side

This interactive simulation demonstrates key database internals like Multi-Version Concurrency Control (MVCC), deadlock detection, and Two-Phase Locking (2PL). Built with Python, SQLite, and a responsive frontend, it provides a visual learning experience for understanding complex database concepts.

Python Backend

Simulates database operations with clean, modular code

SQLite Storage

Local file-based database for persistence and querying

Interactive UI

Responsive design with Tailwind CSS for a smooth experience

Data Visualization

Dynamic charts and graphs for performance analysis

Transaction

BEGIN TRANSACTION;

-- First read operation

SELECT * FROM accounts WHERE id = 1;

-- Update balance

UPDATE accounts SET balance = balance - 100 WHERE id = 1;

-- Read from another table

SELECT * FROM transactions WHERE account_id = 1;

COMMIT;

/* MVCC */
Each transaction sees a snapshot of the database as it existed when the transaction started.
/* 2PL */
Locks are only acquired in the growing phase and only released in the shrinking phase.
